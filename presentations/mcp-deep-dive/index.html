What is MCP?

Model Context Protocol — an open standard for connecting AI agents to tools

Tools — functions the agent can call

Resources — data the agent can read

Prompts — workflow templates the agent can follow

Setup & Connectivity

Multiple transports, connection methods, and install paths

Transport Modes

stdio (default)

  • Local agent integrations
  • Claude Code, Cursor
  • No auth needed
  • One server per agent session

HTTP / SSE

  • Shared deployments
  • Multi-agent access
  • OAuth support (JWT)
  • --host, --port, rate limiting

Connection Options

Direct URL

--database-url redis://localhost:6379

Environment Variable

REDIS_URL=redis://localhost:6379

Profile-Based

--profile prod-cloud --profile staging-enterprise

Profiles support env var substitution, multiple clusters, per-deployment-type defaults

Security & Policy Engine

Every tool annotated. Every action gated. Granular control.

ReadOnly ReadWrite Full

Safety Tiers

TierAllowsUse Case
ReadOnly Only read_only_hint=true tools Exploration, auditing, demos
ReadWrite Reads + non-destructive writes Development, staging
Full All operations including destructive Admin tasks, CI/CD

Default is ReadOnly — safe out of the box

MCP Annotations

Every tool carries machine-readable safety hints


// Read-only: no side effects
database_tool!(read_only, ping, "redis_ping", ...);
//   -> read_only_hint = true, destructive_hint = false

// Write: creates or modifies data
database_tool!(write, set, "redis_set", ...);
//   -> read_only_hint = false, destructive_hint = false

// Destructive: irreversible
database_tool!(destructive, del, "redis_del", ...);
//   -> read_only_hint = false, destructive_hint = true

Annotations are enforced at the macro level — not optional

Policy Files


# ~/.config/redisctl/mcp-policy.toml
tier = "read-only"
deny_categories = ["destructive"]

[cloud]
tier = "read-write"
allow = ["create_database"]
deny = ["delete_subscription"]

[enterprise]
tier = "read-only"

[database]
allow = ["redis_set", "redis_expire"]

[tools]
preset = "essentials"
include = ["redis_ft_search", "redis_ft_create"]

Evaluation Order

  1. Global deny list → DENY
  2. Per-toolset deny list → DENY
  3. Category deny (e.g. "destructive") → DENY
  4. Global allow list → ALLOW
  5. Per-toolset allow list → ALLOW
  6. Per-toolset tier → evaluate
  7. Global tier → evaluate

Deny wins over allow. Specific wins over general.

Raw Tool Protection

Escape-hatch tools are blocked by default

redis_command — arbitrary Redis commands

cloud_raw_api — arbitrary Cloud API calls

enterprise_raw_api — arbitrary Enterprise API calls

Require explicit opt-in via policy file allow list

Existing Skills

Index Advisor

Sample keys → infer field types & cardinality → recommend FT.CREATE schema

Query Tuning

Run FT.EXPLAIN & FT.PROFILE → identify bottlenecks → suggest optimizations

Index A/B Testing

Create variants → run same queries → compare profile results → recommend winner

SKILL.md Format


---
name: index-advisor
description: Analyze dataset and recommend search index schema
---

## Steps

1. Use `redis_keys` to sample keys matching the prefix
2. Use `redis_json_get` to inspect document structure
3. Analyze field types, cardinality, value distributions
4. Recommend field types (TEXT vs TAG vs NUMERIC)
5. Generate ready-to-use `redis_ft_create` call
6. Explain trade-offs (sortable, weight, stemming)

Load custom skills with --skills-dir or REDISCTL_MCP_SKILLS_DIR

The Full Flow


  1. Seed            redis_seed (10k products)
       │
  2. Index           redis_ft_create (define schema)
       │
  3. Search          redis_ft_search ("noise cancelling" @price:[0 200])
       │
  4. Profile         redis_ft_profile (which iterator is slowest?)
       │
  5. Tune            redis_ft_create (variant index)
       │
  6. Compare         redis_ft_info + redis_ft_profile (side by side)
       │
  7. Ship            redis_ft_aliasupdate (zero-downtime swap)

Profile System

Manage credentials for Cloud, Enterprise, and direct Redis connections

Multi-Cluster Configuration


# ~/.config/redisctl/config.toml
default_cloud_profile = "prod"
default_enterprise_profile = "staging"

[profiles.prod]
type = "cloud"
api_key = "${REDIS_CLOUD_API_KEY}"
api_secret = "${REDIS_CLOUD_API_SECRET}"

[profiles.staging]
type = "enterprise"
url = "https://cluster.internal:9443"
username = "admin"
password = "${RE_PASSWORD}"

[profiles.local]
type = "database"
url = "redis://localhost:6379"
